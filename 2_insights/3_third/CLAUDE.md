# Third(3번째 번호) 인사이트

정렬된 당첨번호 중 3번째 번호(ball3/ord3) 분석 및 선택

## 실행

```bash
python export_result.py
```

- **입력**: result/result.csv (ord1, ord2, ord6 있음)
- **출력**: result/result.csv (ord3 추가, 복수 추천시 행 복제)

## ord3 선택 로직

### 범위 제약
- `ord2 + 1 ≤ ord3 ≤ ord6 - 3` (ord4, ord5 자리 확보)

### 점수화 요소

| 요소 | 점수 | 설명 |
|------|------|------|
| ball3 빈도 | ×2 | 해당 번호의 ball3 출현 빈도 |
| ball2-ball3 간격 | ÷2 | 간격별 통계 빈도 |
| (ball1,ball2)→ball3 매칭 | +20 | 조합별 최빈 ball3 일치 |
| 최빈 구간 10-19 | +12 | 45.4% 집중 |
| 최빈 구간 20-29 | +8 | 36.7% 집중 |
| 연속수 (gap=1) | +5 | 10.8% 비율 |
| 원핫 ord3 비트 빈도 | +0~23 | 포지션별 비트 활성화 빈도 |
| 핫 비트 | +10 | {29,17,27,3,25,1,19,39,4,31,37,33,10,2,32} |
| 콜드 비트 | -5 | {41,43,8,14,34,26,22,44,20,7,30,15,11,40,6} |

## Range 반영 (80/20 전략)

### 점수화 요소 추가
| 요소 | 점수 | 설명 |
|------|------|------|
| 80% 잘 나오는 구간 (1,2) | ×0.3 | 최대 13점 |
| 20% 나올 가능성 구간 (0,3) | ×0.5 | 최대 6점 |
| 새 구간 추가 (4개 목표) | +10 | 51.2% 최빈 |
| 연속/인접 구간 | +5 | 57.5% |

### 4개 구간 사용이 최적
- 3개 구간 → 4개: +10점
- 4개 유지: +5점
- 5개: +3점
